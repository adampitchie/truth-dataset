# Analyzing the Dataset

A guide to data-science patterns for exploring this dataset of Trump's Truth
Social posts — a large collection of short, timestamped texts (with image
descriptions and video transcripts). The emphasis is on understanding **what is
said**, **when**, and especially **what is distinctive about specific periods**.

Analysis assumes you've generated `data/dataset.parquet` by running
[`compile.py`](compile.py)
(see [README](README.md#generate-the-parquet-file-do-this-first)) and loaded it:

```python
import pandas as pd
df = pd.read_parquet("data/dataset.parquet")
df = df.set_index("timestamp").sort_index()
```

> **Where `all_text` comes from:** the convenience columns used throughout this
> guide — `all_text`, `image_descriptions`, `video_transcripts`, `n_images`,
> `n_videos` — do **not** exist in the raw `dataset.jsonl`. They are generated by
> `compile.py` and baked into the Parquet file, so loading Parquet is all you
> need. `all_text` is the post text + image descriptions + video transcripts
> joined together, so text analysis covers media-only posts too.

The three layers below build on each other: find *when* things happen, then
*what* is being said, then combine the two to characterize important periods.

---

## 1. Temporal patterns — find the clusters

Start with simple aggregates. These reveal structure quickly, before any heavier NLP.

| Pattern | What it reveals | Tooling |
|---|---|---|
| **Volume time series** | Resample by day/week/month → high-activity periods become visible | `df.resample("D").size()` |
| **Burst / anomaly detection** | Days statistically above baseline (rolling mean + z-score, or Kleinberg bursts) | rolling z-score, `pybursts` |
| **Change-point detection** | Dates where the *baseline* shifts (cadence, volume), not just spikes | `ruptures` (PELT) |
| **Cadence heatmaps** | Hour-of-day × day-of-week rhythm; odd-hour bursts | seaborn / Plotly heatmap |
| **Event overlay** | Volume with vertical markers at known dates → see how posting responds to specific events | annotated time series |

```python
# Daily volume with a rolling-z-score burst flag
daily = df.resample("D").size().rename("posts").to_frame()
daily["z"] = (daily["posts"] - daily["posts"].rolling(30).mean()) \
             / daily["posts"].rolling(30).std()
bursts = daily[daily["z"] > 3]
```

The output of this layer is a set of **date windows** to explore textually.

---

## 2. Text patterns — understand the content at scale

Use the `all_text` column (post text + image descriptions + transcripts) so
media-only posts are included.

| Pattern | Purpose | Tooling |
|---|---|---|
| **TF-IDF + top terms / n-grams** | First look at the most frequent vocabulary and phrases | `sklearn` `TfidfVectorizer` |
| **Topic modeling** | Discover latent themes. Classic: LDA/NMF. Modern & better for short text: **BERTopic** | `gensim`, `BERTopic` |
| **Embeddings + clustering** | Encode posts → semantic groups (HDBSCAN/k-means) and **semantic search** | `sentence-transformers`, `hdbscan` |
| **Named Entity Recognition** | People/orgs/places mentioned → frequency & co-occurrence | `spaCy` |
| **Sentiment / emotion** | Tone per post → trend over time | `VADER`, transformer classifiers |

```python
# Embed once, cache, reuse everywhere downstream
from sentence_transformers import SentenceTransformer
import numpy as np

model = SentenceTransformer("all-MiniLM-L6-v2")
emb = model.encode(df["all_text"].tolist(), show_progress_bar=True)
np.save("data/embeddings.npy", emb)   # reuse for topics, clustering, search
```

> **Performance note:** embedding is the computationally expensive step. At ~37k
> rows it runs in minutes on a laptop. Cache the vectors — everything downstream
> is then fast.

---

## 3. The intersection — "what's distinctive about *this* period"

The highest-value layer, and the one most often skipped.

- **Term-frequency-over-time** — track keywords as time series
  (`mentions of "tariffs" per week`); shows when a topic enters or exits.
- **Dynamic topic modeling** — BERTopic's `topics_over_time` shows how each
  theme's prevalence changes across your date windows.
- **Distinctive-terms-per-window (key technique)** — for a burst window vs. the
  baseline corpus, find *over-represented* words. Don't use raw counts — use the
  **weighted log-odds ratio with an informative Dirichlet prior**. It surfaces
  what makes a period distinctive without being distorted by common or rare words.
- **scattertext** — visualize word usage between two periods (e.g. pre/post a
  change-point).
- **Per-cluster exemplars** — for each temporal/topic cluster, pull the most
  representative posts and *read them*. Always ground statistics in real text.

---

## 4. Visualization — seeing beyond the text

Charts are very useful here: temporal clusters, topic drift, and outliers are
far easier to *see* than to read in tables. Pick the library by goal:

| Library | Best for | Style |
|---|---|---|
| **Plotly** | Interactive temporal exploration — zoom into time windows, hover individual posts, heatmaps, scatter maps | Interactive, web-ready |
| **Altair** | Clean declarative statistical charts, linked/brushable views, bar charts, distributions | Elegant, concise |

### A chart catalog mapped to insight

| Chart | What it surfaces | Library |
|---|---|---|
| **Volume line/area over time** | Overall activity, bursts, quiet periods | Plotly / Altair |
| **Calendar heatmap** (GitHub-style) | Daily posting intensity across years at a glance | `plotly`, `calplot` |
| **Hour × weekday heatmap** | Behavioral rhythm; unusual-hour activity | Plotly |
| **Keyword-frequency-over-time (multi-line)** | When specific topics increase or decline in frequency | Plotly / Altair |
| **Topic stream / stacked area** | Composition of themes shifting over time | Plotly |
| **UMAP embedding scatter** (colored by topic or date) | The "map" of the dataset; semantic clusters and outliers | Plotly |
| **Distinctive-terms bar chart** (per window) | What makes a period unique | Altair |
| **Entity co-occurrence network** | Who/what is mentioned together | `pyvis`, `networkx` |
| **Sentiment-over-time band** | Sentiment trends and abrupt shifts in tone | Plotly / Altair |

```python
# Interactive daily-volume timeline (zoomable to any burst window)
import plotly.express as px

daily = df.resample("D").size().rename("posts").reset_index()
fig = px.area(daily, x="timestamp", y="posts",
              title="Daily posting volume")
fig.update_xaxes(rangeslider_visible=True)   # drag to zoom into clusters
fig.show()
```

```python
# Embedding map: 2-D projection colored by year
import umap, plotly.express as px
import numpy as np

emb = np.load("data/embeddings.npy")
xy = umap.UMAP(n_neighbors=15, min_dist=0.1).fit_transform(emb)
plot_df = df.reset_index()[["timestamp", "all_text"]].copy()
plot_df["x"], plot_df["y"] = xy[:, 0], xy[:, 1]
plot_df["year"] = plot_df["timestamp"].dt.year
px.scatter(plot_df, x="x", y="y", color="year",
           hover_data={"all_text": True, "x": False, "y": False},
           title="Semantic map of posts").show()
```

## Recommended workflow

```
1. Compute temporal aggregates (volume, cadence) → identify candidate date windows
2. Detect bursts / change-points              → formalize those windows
3. Embed all posts once (cache the vectors)   → reuse downstream
4. BERTopic for global themes + topics_over_time
5. Per important window: distinctive-terms (log-odds) + read exemplars
6. Track key entities/keywords as time series across the full span
7. Visualize each step — charts make patterns, structure, and outliers observable
```

## Generating new research directions

### Prompting Claude for progressively deeper research ideas

You don't have to think up every analysis yourself — use Claude (see the
README's [no-code section](README.md#exploring-the-dataset-with-claude-no-code))
to brainstorm and then explore ideas. The prompts below progress incrementally from
standard descriptive work to research-based methods.

**1. Descriptive — establish the baseline**

- *"Profile the dataset: posting volume over time, post-length distribution,
  share of media-only posts, and busiest hours. Give me the headline facts."*

**2. Comparative — contrast slices**

- *"Compare 2022 vs 2025 on volume, average length, media use, and top terms.
  What changed, and which differences look meaningful?"*

**3. Inferential — test for significance**

- *"Is the difference in posting rate between weekdays and weekends statistically
  significant? Choose an appropriate test, state the hypotheses, and report the
  effect size with caveats."*

**4. Modeling — surface latent structure**

- *"Run BERTopic on `all_text`, label the topics, and chart how the prevalence
  of the top 10 changes over time."*
- *"Cluster posts by embedding and characterize each cluster with distinctive
  terms (weighted log-odds) plus three exemplar posts."*

**5. Relational & temporal dynamics**

- *"Build a co-occurrence network of named entities; identify community structure and high-centrality entities per year."*
- *"Model the daily post count as a time series — test for autocorrelation,
  seasonality, and whether bursts cluster around dates I provide."*

**6. Causal & predictive**

- *"Design an event study around [date]: define pre/post windows and a
  counterfactual baseline, then measure the shift in topic mix and sentiment.
  Identify the limitations and validity concerns."*
- *"Propose a falsifiable hypothesis linking posting behavior to an external
  signal I supply (e.g. a market index or news volume), specify an identification
  strategy, and name the likely confounds."*

**Meta-prompts that generate the ideas for you**

- *"Act as a computational social scientist. Given this dataset's schema, propose
  12 research questions ranked from descriptive to causal. For each, give the
  method, the column(s) it uses, and the main confound."*
- *"Take the previous idea and make it more sophisticated three times in a row —
  each time naming the limitation it overcomes and the technique it adds."*
- *"What analyses does this data support that I haven't asked about? Rank them by
  novelty and feasibility."*

> **Tip:** ask Claude to **state assumptions, limitations, and potential sources of invalidity** for any non-trivial claim.

### Seeking out methodologies beyond this guide

This doc details established, empirically validated methods — but it's deliberately not exhaustive. The best technique for *your* question may
be one not listed here, or borrowed from another discipline entirely. Use Claude
to actively search for it. Ask it to **search the web and cite sources** so
suggestions are current and verifiable rather than from memory.

**Find the state of the art for a specific task**

- *"What are the current state-of-the-art methods for [short-text topic modeling
  / burst detection / stance detection] as of this year? Compare them to what
  this guide uses and note the tradeoffs. Cite sources."*

**Borrow from other disciplines**

- *"What techniques from econometrics, signal processing, or network
  science could apply to a time series of social-media posts? Give one
  specific method from each field and how I'd apply it to this dataset."*

**Match methods to this data's specific quirks**

- *"My data is ~37k timestamped posts with mixed authorship and
  multimodal media. What specialized methods exist for these exact properties
  that a generic NLP pipeline would miss?"*

**Critique and upgrade the current plan**

- *"Here is my current approach: [paste a section of this guide]. Act as a
  skeptical methods reviewer — what's outdated, naive, or fragile, and what newer
  or more rigorous technique would a researcher use instead?"*

**Keep novelty honest**

- *"For each method you suggested, name its key assumption, when it fails, and a
  simple baseline to benchmark it against, so I don't adopt complexity unecessarily."*

> **Tip:** always ask for **citations or links** and a **simple baseline**. A
> novel method isn't better until it beats the simple version on *your* data —
> have Claude run that comparison before you commit to it.
